10分钟解决跨域

并不是众所周知…默认情况下, 服务器是不允许跨域的
假如直接手写一个服务器, 再通过ajax 申请数据, 通过会得到这样的结果:

翻译一下: 现在8080端口的 getNews页面上没有允许的请求源, 所以3000端口的请求被拒绝了

Q: why?
A: 因为服务器没有允许请求数据的源(origin), 故无法向服务器要数据

Q: 那别人家的网站(孩子)怎么行?
A: 他们使用了跨域

Q: 跨域是啥子哟
A: 那就要从什么是同域开始了, 同域意味着以下3点:

  • 协议相同 http 与 https 无法相互获取数据
  • 域名相同 a.com 与 b.com 无法相互获取数据, 即使通过修改 DNS 都指向127.0.0.1也不行
  • 端口相同

客户端与服务器不在同域时, 就得使用跨域来获取数据了
目前跨域有两种方案:

1. JSONP 方案

回想一下我们引用各种 js 文件如 jquery, font-aswsome, 说到底也是向服务器去要数据, 这个时候我们并没有受到同源限制, 即 script 标签不受同源策略限制.
这就是 jsonp 的由来, 全称 json with padding. 带有补丁的 json, 应用方式有点小小的 hack: 创建 script 标签去获取数据(再删除), 标签中带有一个callback 来处理数据.
这么 hack 的方式总有点麻烦…我也懒得写了:
5 分钟彻底明白 JSONP

2. CORS 方案

CORS 全称 Cross-Origin Resource Sharing, 即跨域资源共享. 比起 hacky 的 jsonp, 这种方法可是优雅多了:

  • 实现方法:在服务器端设置允许的响应头(response header)
  • 除了一行 setHeader 跟同域请求几乎一抹一眼

Q: 这能不能设置多个啊?
A: 可以~

Q: 这才两个
A: 那给你个无限个的:

Q: 我不能自己决定
A: 你咋问题这么多,最后一个了:

html 的 url 与服务器原会比较
ajax 的 url 指向服务器
服务器可设置响应头来允许对应的 html 的 url 跨域

本地文件通过 ajax 获取本地服务器数据

写项目的时候经常会有这样的操作:

  • 需要通过 ajax 向服务器获取数据
  • 搭建一个本地静态服务器
  • 顺手打开了文件夹中的 html文件
  • 报错（＞﹏＜）:

查了一下发现, chrome 认为这是不安全的, 默认阻止了file 协议使用 ajax(XMLHttpRequest对象)
…然而我用 file 协议打开 html, ajax 指向 easy-mock 却没有任何问题
解决办法?

  • 换个浏览器? but…chrome 那么好用, 还是舍不得
  • 通过终端来 disable 掉 web secure? but…听起来感觉不是很安全.
  • 换个IDE? but…atom 随意定制界面优雅,实在太好用了不舍得
  • 加个插件? atom 上的 atom-live-server 是个很好的选择
    • 安装
    • 打开任意窗口 command + shift + p
    • 输入 atom-live-server 随便选个端口
    • 打开浏览器, 输入localhost:端口号
    • 选择刚刚的 html 文件
    • 搞定!
